# Функциональное описание сайта IT-Eco-For-SP

## Общее описание

**IT-Eco-For-SP** — образовательная экосистема для студентов с ИИ-ассистентом, интеграцией с Codeforces/AtCoder и системой рейтингов.

**Стек:** Next.js 15, React 19, TypeScript, SurrealDB, NextAuth.js

---

## Главная страница — информационное наполнение

### 1. Шапка (Header)

Фиксированная навигационная панель вверху экрана.

```
┌──────────────────────────────────────────────────────────────────┐
│  [БЦСП]    Главная    База знаний    Календарь    ИИ Ассистент  │
│                                            Профиль    Вход      │
└──────────────────────────────────────────────────────────────────┘
```

Состав:
- **Логотип «БЦСП»** — текстовый, цвет `#257AB3`. Визуальный якорь бренда.
- **Навигация** — ссылки на основные разделы. Ссылки «ИИ Ассистент» и «Профиль» видны только авторизованным пользователям. Ссылка «Вход» — только неавторизованным.

### 2. Блок для новых участников: «Что такое спортивное программирование?»

Этот блок — первое, что видит человек, никогда не слышавший о спортпроге. Его задача — объяснить суть, заинтересовать и подтолкнуть к регистрации.

#### Заголовок
> **Что такое спортивное программирование?**

#### Основной текст
> Спортивное программирование — это соревнования, на которых участники решают алгоритмические задачи за ограниченное время.
>
> Каждая задача — это головоломка: нужно придумать алгоритм, написать код и уложиться в лимиты по времени и памяти.
> Побеждает тот, кто решит больше задач быстрее и без ошибок.

#### Зачем это нужно? (блок преимуществ)

Оформляется в виде карточек-преимуществ (3–4 карточки в ряд):

| Карточка | Иконка | Текст |
|----------|--------|-------|
| **Прокачка алгоритмов** | 🧠 | Ты научишься решать задачи, которые дают на собеседованиях в Яндекс, Google, TikTok и другие топ-компании |
| **Карьерные перспективы** | 💼 | Высокий рейтинг на Codeforces — это строчка в резюме, которая открывает двери. Многие компании напрямую просматривают профили на CF |
| **Сообщество и нетворкинг** | 🤝 | Тысячи единомышленников, командные тренировки, олимпиады — ты попадаешь в среду сильных разработчиков |
| **Это бесплатно и доступно** | 🌍 | Начать можно прямо сейчас. Нужен только компьютер и интернет. Большинство площадок полностью бесплатны |

#### Призыв к действию (CTA)

Под блоком преимуществ — заметная кнопка:

> **[Начать путь в спортпроге →]**
>
> Ссылка на `/auth/signup` (регистрация)

## Страницы

### 1. Главная страница — `/home`

**Для кого:** Все посетители (не требует авторизации)

**Содержимое:**

- 1 - 3 блоки - были описаны выше
- **Блок предстоящих событий (UpcomingEvents)** — карточки ближайших соревнований
  - Данные загружаются с сервера (`/api/events`)
  - Каждая карточка содержит: название контеста, платформу (Codeforces / AtCoder), дату начала, дату окончания, ссылку на регистрацию

**Карточка события содержит:**
- Название контеста
- Бейдж платформы (цветной)
- Дата начала и окончания
- Кнопка/ссылка **«Перейти к контесту»** — открывает страницу соревнования на внешней платформе

### 2. База знаний — `/base`

**Для кого:** Все посетители

**Содержимое:**
- Заглушка с текстом **«Информация готовится»**
- Страница планируется как раздел с учебными материалами

### 3. Календарь соревнований — `/calendar`

**Для кого:** Все посетители

**Содержимое:**

- **Календарная сетка** — таблица текущего месяца с днями недели
- **Навигация по месяцам:**
  - Кнопка **«←»** (предыдущий месяц)
  - Кнопка **«→»** (следующий месяц)
  - Отображение текущего месяца и года в заголовке
- **Ячейки дней:**
  - Номер дня
  - Если на этот день приходится соревнование — отображается его название с цветным бейджем платформы
  - Название контеста кликабельно (ссылка на внешнюю страницу)

**Загрузка данных:** Все контесты загружаются с `/api/contests/all`

### 4. ИИ-ассистент — `/chat`

**Для кого:** Только авторизованные пользователи

**Содержимое:**

- **Окно чата** — область переписки с ИИ-ассистентом
- **Поле ввода** — текстовое поле для отправки сообщений
- **Кнопка «Отправить»** — отправка сообщения в чат

**Как работает:**
1. Пользователь вводит вопрос или запрос в поле ввода
2. Нажимает **«Отправить»** (или Enter)
3. Сообщение отображается в окне чата
4. ИИ-ассистент отвечает с анимацией «печати» (посимвольное появление текста)
5. Ответы генерируются моделью **Qwen3** через RouterAI API с использованием RAG-контекста (поиск по базе новостей и контестов)

**RAG-контекст:** ИИ-ассистент имеет доступ к актуальным новостям проекта и данным о соревнованиях, хранящимся в базе данных.

### 5. Панель управления — `/dashboard`

**Для кого:** Только авторизованные пользователи

**Содержимое:**

- **Информация о пользователе:**
  - Полное имя
  - Объединённый рейтинг БЦСП
- **Блок подключения Codeforces (CodeforcesConnect):**
  - Поле ввода **handle Codeforces**
  - Кнопка **«Подключить»**
  - Пошаговый процесс подключения:
    1. Ввести handle Codeforces
    2. Получить уникальный код верификации
    3. Вставить код в поле «First Name» профиля Codeforces
    4. Нажать кнопку **«Подтвердить»**
  - После успешной верификации — редирект на `/profile`
  - Кнопка **«Отключить»** — отвязывает аккаунт Codeforces

---

### 6. Профиль пользователя — `/profile`

**Для кого:** Только авторизованные пользователи

**Содержимое:**

#### Блок личной информации
- **Полное имя** (отображается)
- **Email** (отображается, нередактируемый)
- **Телефон** (отображается)
- **Рейтинг БЦСП** (отображается)

#### Форма редактирования профиля
- Поле **«Имя»** — редактирование полного имени
- Поле **«Телефон»** — редактирование номера телефона
- Поле **«Новый пароль»** — изменение пароля (необязательное)
- Кнопка **«Сохранить»** — сохраняет изменения

#### Интеграция с Codeforces
- Отображение подключённого handle (если подключён)
- Поле ввода **handle Codeforces** + кнопка **«Подключить»** (если не подключён)
- Кнопка **«Отключить»** — отвязывает аккаунт
- Процесс верификации аналогичен `/dashboard`

#### Интеграция с AtCoder
- Отображение подключённого handle (если подключён)
- Поле ввода **handle AtCoder** + кнопка **«Подключить»** (если не подключён)
- Кнопка **«Отключить»** — отвязывает аккаунт
- Процесс верификации: код размещается в поле **«Affiliation»** профиля AtCoder

#### Таблица истории соревнований
- Объединённая таблица контестов Codeforces + AtCoder, отсортированная по дате
- **Колонки таблицы:**
  - Дата
  - Название контеста
  - Платформа (Codeforces / AtCoder)
  - Рейтинг до
  - Рейтинг после
  - Изменение рейтинга (подсвечивается цветом: зелёный — рост, красный — падение)
  - Место в рейтинге

**Фильтры таблицы:**
- **По дате** — выбор диапазона (дата начала — дата окончания)
- **По платформе** — Codeforces / AtCoder / Все
- **По месту** — диапазон мест (от — до)
- **Сортировка** — по изменению рейтинга (возрастание / убывание)

**Раскрытие контеста:**
- Клик по строке контеста раскрывает список решённых задач
- Данные загружаются из внешних API (Codeforces API / AtCoder Problems API)
- Для каждой задачи отображается: название, статус (решена/не решена), ссылка на задачу

### 7. Вход — `/auth/signin`

**Для кого:** Только неавторизованные пользователи

**Содержимое:**

- Поле **«Email»** — ввод email-адреса
- Поле **«Пароль»** — ввод пароля
  - Кнопка **«Показать/скрыть пароль»** (иконка глаза)
- Кнопка **«Войти»** — выполняет вход
- Ссылка **«Зарегистрироваться»** — переход на `/auth/signup`

**Обработка ошибок:**
- Неправильный логин/пароль — сообщение об ошибке
- Email не подтверждён — перенаправление на страницу верификации
- После успешного входа — редирект на `/home`

---

### 8. Регистрация — `/auth/signup`

**Для кого:** Только неавторизованные пользователи

**Содержимое:**

- Поле **«Полное имя»**
- Поле **«Email»**
- Поле **«Пароль»**
- Поле **«Подтверждение пароля»**
- Кнопка **«Зарегистрироваться»**
- Ссылка **«Уже есть аккаунт?»** — переход на `/auth/signin`

**Валидация:**
- Проверка совпадения паролей
- Проверка формата email
- После успешной регистрации — редирект на `/auth/verify-email`

**Что происходит на сервере:**
1. Создаётся запись пользователя в SurrealDB с `is_verified: false`
2. Генерируется 6-значный код верификации
3. Код отправляется на указанный email через Nodemailer

---

### 9. Подтверждение email — `/auth/verify-email`

**Для кого:** Пользователи, прошедшие регистрацию, но не подтвердившие email

**Содержимое:**

- Поле **«Email»** — email, указанный при регистрации
- Поле **«Код верификации»** — 6 цифр из письма
- Кнопка **«Подтвердить»** — проверяет код
- Кнопка **«Отправить код повторно»** — отправляет новый код на email

**После успешной верификации:**
- Пользователь получает `is_verified: true`
- Через 3 секунды — автоматический редирект на `/auth/signin`
