# Новые реализованные функции для дизайна

Этот документ описывает функции, которые уже есть в проекте, но не вошли в основной `docs/FUNCTIONAL_DESCRIPTION.md`. Основной фокус — административный и тренерский контуры, по которым дизайн ещё не сделан.

Часть frontend-реализации сейчас техническая/черновая, поэтому для дизайнеров важно спроектировать полноценные интерфейсы, состояния и пользовательские сценарии.

---

## 1. Панель администратора — `/admin`

**Для кого:** Только пользователи с ролью `admin`

**Назначение:** Центральная точка входа в административные функции системы.

**Содержимое:**

- Заголовок **«Панель администратора»**
- Подзаголовок: управление системой и пользователями
- Карточки перехода в основные разделы:
  - **Управление пользователями** → `/admin/users`
  - **Корректировка кармы** → `/admin/karma`
  - **Управление контестами** → `/coach/contests`
- Состояния:
  - загрузка доступа
  - отказ в доступе с возвратом на dashboard

**Важно для дизайна:**

- Это рабочая панель, не маркетинговый экран. Нужен плотный dashboard-стиль: понятные карточки действий, быстрый обзор разделов, минимум декоративности.
- Сейчас в коде есть блок **«Быстрые действия»**, но кнопки экспорта и настроек системы пока не имеют отдельной реализованной логики. Их можно заложить как future-состояние или не делать акцент.

## 2. Управление пользователями — `/admin/users`

**Для кого:** Только `admin`

**Назначение:** Полный административный контроль аккаунтов: просмотр, создание, редактирование, блокировка, удаление и назначение ролей.

### Таблица пользователей

Отображает всех пользователей системы.

**Колонки:**

| Колонка          | Содержимое                                       |
| ---------------- | ------------------------------------------------ |
| Email            | email пользователя                               |
| Имя              | `full_name`, если заполнено                      |
| Роль             | бейдж: Гость / Участник / Тренер / Администратор |
| Верификация      | подтверждён email или нет                        |
| Статус           | активен / заблокирован                           |
| Рейтинг          | рейтинг БЦСП и карма                             |
| Дата регистрации | дата создания аккаунта                           |
| Действия         | редактировать, удалить                           |

### Создание пользователя

Открывается через кнопку **«Создать пользователя»**.

**Поля формы:**

- Email
- Пароль
- Имя
- Телефон
- Роль: `guest`, `user`, `coach`, `admin`
- Чекбокс **«Email подтверждён»**
- Чекбокс **«Аккаунт заблокирован»**

**Поведение:**

1. Администратор заполняет форму.
2. Система проверяет обязательные поля и уникальность email.
3. Пароль хешируется на сервере.
4. Пользователь создаётся сразу с выбранной ролью и статусом.

### Редактирование пользователя

Открывается через действие редактирования в строке таблицы.

**Редактируемые поля:**

- Email
- Имя
- Телефон
- Роль
- Верификация email
- Блокировка аккаунта

**Защитное правило:**

- Нельзя удалить, заблокировать или понизить последнего активного администратора.
- Если пользователь заблокирован, он не может войти в систему.

### Матрица прав роли

В модальном окне создания/редактирования показывается превью прав выбранной роли.

**Права:**

| Право                  | Что означает                        |
| ---------------------- | ----------------------------------- |
| Лендинг и маркетинг    | просмотр публичных страниц          |
| Глобальный рейтинг     | просмотр рейтинговых данных         |
| Календарь соревнований | просмотр предстоящих событий        |
| AI-ассистент           | доступ к `/chat`                    |
| Участие в контестах    | пользовательский сценарий участия   |
| Личный dashboard       | доступ к личному кабинету           |
| Управление контестами  | создание и настройка соревнований   |
| Аналитика групп        | просмотр тренерской аналитики       |
| Управление аккаунтами  | административный CRUD пользователей |
| Корректировка кармы    | ручное изменение кармы              |

### Системное поведение

- Создание, изменение и удаление пользователя пишется в `admin_audit_logs`.
- Отдельного экрана просмотра audit log пока нет, но данные уже сохраняются на сервере.

**Состояния для дизайна:**

- пустой список пользователей
- ошибка загрузки
- успешное создание/обновление/удаление
- ошибка: email уже существует
- ошибка: нельзя изменить последнего активного администратора
- подтверждение удаления

## 3. Корректировка кармы — `/admin/karma`

**Для кого:** Только `admin`

**Назначение:** Ручное изменение кармы пользователя, например за офлайн-активности или административную корректировку.

### Основной сценарий

1. Администратор нажимает **«Скорректировать карму»**.
2. Открывается форма.
3. Администратор выбирает пользователя из списка.
4. Вводит изменение кармы:
   - положительное число — увеличить
   - отрицательное число — уменьшить
5. Указывает причину.
6. Нажимает **«Применить»**.
7. Система показывает успешное изменение в формате: было → стало.

### Поля формы

- Пользователь
- Изменение кармы
- Причина

### История изменений

На странице предусмотрена таблица истории.

**Колонки:**

| Колонка   | Содержимое                            |
| --------- | ------------------------------------- |
| Email     | пользователь, которому изменили карму |
| Изменение | число с плюсом или минусом            |
| Причина   | текст причины                         |
| Дата      | дата изменения                        |

**Состояния для дизайна:**

- форма скрыта / раскрыта
- пустая история
- положительное изменение
- отрицательное изменение
- ошибка сохранения
- успешное изменение

## 4. Тренерская — `/coach`

**Для кого:** `coach` и `admin`

**Назначение:** Центральная рабочая зона тренера.

**Содержимое:**

- Заголовок **«Тренерская»**
- Разделы:
  - **Мероприятия** → `/coach/events`
  - **Группы** → `/coach/groups`
- Состояния:
  - загрузка доступа
  - отказ в доступе

**Важно для дизайна:**

- Это рабочий экран для регулярного использования тренером.
- Нужны быстрые переходы к управлению мероприятиями, группами и аналитикой.
- Администратор имеет доступ к тренерским разделам как роль более высокого уровня.

## 5. Управление мероприятиями — `/coach/events`

**Для кого:** `coach` и `admin`

**Назначение:** Создание и управление учебными мероприятиями/контестами с публичной или приватной видимостью.

### Список мероприятий

Каждая карточка мероприятия содержит:

- Название
- Бейдж платформы: Codeforces / AtCoder / Своя ссылка / Другое
- Бейдж статуса: предстоящий / активный / завершён / отменён
- Бейдж видимости: public / private
- Дату начала
- Дату окончания
- Внешнюю ссылку **«Перейти к контесту»**
- Действия:
  - редактировать
  - удалить
  - синхронизировать результаты, если мероприятие завершено и приватное

### Создание и редактирование мероприятия

Форма открывается по кнопке **«Создать мероприятие»** или действию редактирования.

**Поля формы:**

| Поле           | Тип                          |
| -------------- | ---------------------------- |
| Название       | текст                        |
| Описание       | textarea                     |
| Платформа      | select                       |
| Статус         | select                       |
| Начало         | datetime                     |
| Окончание      | datetime                     |
| Внешняя ссылка | url                          |
| Видимость      | radio: публичное / приватное |

### Публичное мероприятие

Видно всем пользователям в календаре и списках событий.

### Приватное мероприятие

Видно только назначенным пользователям.

**Способы назначения:**

- напрямую выбрать пользователей
- выбрать группы
- совместить оба способа

**Поведение выбора:**

- Если выбрана группа, её участники не дублируются в ручном выборе пользователей.
- Для приватного мероприятия обязательно указать хотя бы одного пользователя или одну группу.

### Синхронизация результатов

Для завершённых приватных мероприятий есть действие **«Синхронизировать»**.

**Смысл действия:**

- система собирает фактический состав участников
- создаёт snapshot участников
- обновляет результаты/участие по внешней платформе

**Состояния для дизайна:**

- пустой список мероприятий
- создание
- редактирование
- удаление с подтверждением
- public/private карточки
- ошибка сохранения
- ошибка синхронизации
- успешная синхронизация

## 6. Простое управление контестами — `/coach/contests`

**Для кого:** `coach` и `admin`

**Назначение:** Более простой/старый интерфейс управления контестами. В проекте он ещё существует и доступен из админской панели.

**Содержимое:**

- Заголовок **«Управление контестами»**
- Кнопка **«Создать контест»**
- Форма создания:
  - название
  - платформа: Codeforces / AtCoder / Другая
  - время начала
  - длительность в минутах
- Список существующих контестов
- Карточка контеста:
  - название
  - статус: предстоящий / активный / завершённый
  - платформа
  - начало
  - длительность
  - действия: редактировать, удалить

**Важно для дизайна:**

- Этот раздел пересекается с `/coach/events`.
- Для дизайнеров стоит решить, должен ли он остаться отдельным экраном или быть объединён с управлением мероприятиями.

## 7. Группы тренера — `/coach/groups`

**Для кого:** `coach` и `admin`

**Назначение:** Управление учебными группами.

### Список групп

**Содержимое:**

- Заголовок **«Группы»**
- Кнопка **«Создать группу»**
- Поле поиска по названию группы
- Карточки групп

**Карточка группы содержит:**

- Название
- Описание, если заполнено
- Действия:
  - открыть состав группы
  - открыть аналитику группы
  - архивировать группу

### Создание группы

Форма содержит:

- Название
- Описание

После создания текущий тренер автоматически становится владельцем группы.

### Архивация группы

Группа не удаляется физически, а получает статус `is_archived = true` и исчезает из активного списка.

### Ограничение доступа

- `admin` видит все группы.
- `coach` видит только группы, где он назначен тренером/владельцем.
- `user` видит только группы, где он состоит, но тренерский интерфейс ему недоступен.

**Состояния для дизайна:**

- пустой список групп
- поиск без результатов
- ошибка загрузки
- создание группы
- подтверждение архивации

## 8. Состав группы — `/coach/groups/[id]`

**Для кого:** `coach` группы и `admin`

**Назначение:** Управление участниками конкретной группы.

### Верхняя зона

- Ссылка назад к списку групп
- Название группы
- Кнопка **«Аналитика»** → `/coach/groups/[id]/analytics`

### Состав группы

Таблица текущих участников.

**Колонки:**

| Колонка  | Содержимое        |
| -------- | ----------------- |
| Имя      | имя участника     |
| Email    | email участника   |
| Действие | удалить из группы |

### Добавление участников

**Содержимое блока:**

- Поиск по имени/email
- Таблица пользователей, которых ещё нет в группе
- Чекбоксы выбора нескольких пользователей
- Кнопка **«Добавить выбранных»**

**Поведение:**

1. Тренер ищет пользователей.
2. Отмечает одного или нескольких.
3. Нажимает **«Добавить выбранных»**.
4. Пользователи появляются в составе группы.

**Ограничение данных:**

- Для тренера список доступных пользователей ограничен участниками его групп.
- Администратор может работать шире.

**Состояния для дизайна:**

- группа без участников
- нет пользователей для добавления
- выбран один пользователь
- выбран список пользователей
- удаление участника с подтверждением
- ошибка доступа

## 9. Аналитика группы — `/coach/groups/[id]/analytics`

**Для кого:** `coach` группы и `admin`

**Назначение:** Быстрый обзор состояния учебной группы.

### Метрики

Карточки верхнего уровня:

| Метрика              | Что показывает                                       |
| -------------------- | ---------------------------------------------------- |
| Участников           | количество пользователей в группе                    |
| Средний BSCP рейтинг | среднее значение рейтинга участников                 |
| Событий назначено    | количество приватных мероприятий, назначенных группе |
| Событий завершено    | количество завершённых мероприятий группы            |

### Платформы

Блок распределения подключённых аккаунтов:

| Категория     | Что означает                           |
| ------------- | -------------------------------------- |
| Codeforces    | подключён только Codeforces            |
| AtCoder       | подключён только AtCoder               |
| Обе платформы | подключены Codeforces и AtCoder        |
| Не подключены | нет верифицированных внешних аккаунтов |

### Действия

- Кнопка **«Обновить»** — повторная загрузка аналитики
- Ссылка назад к составу группы

**Ограничение доступа:**

- Тренер видит аналитику только своих групп.
- Администратор видит аналитику любой группы.

**Состояния для дизайна:**

- загрузка
- нет данных
- ошибка доступа
- ошибка загрузки
- значения `0`
- значение среднего рейтинга отсутствует (`—`)

## 12. Общие состояния для админского и тренерского контуров

Для всех рабочих экранов нужны единые паттерны:

- загрузка данных
- отказ в доступе
- пустой список
- ошибка API
- успешное сохранение
- подтверждение удаления/архивации
- модальное окно создания/редактирования
- inline-валидация обязательных полей
- длинные таблицы и списки с горизонтальным/вертикальным скроллом
- адаптация под мобильный экран, но приоритет — удобство desktop/tablet
