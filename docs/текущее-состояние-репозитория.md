## 2. Назначение продукта

Образовательная экосистема для студентов: ИИ-ассистент (RAG, несколько провайдеров через AI SDK), интеграции **Codeforces** и **AtCoder**, календарь соревнований, профиль и рейтинги, email-верификация, роли **admin** / **coach** / **user** (NextAuth + middleware).

## 8. Краткий вывод

Репозиторий — **монолитное Next.js-приложение** с SurrealDB, NextAuth и развитой доменной логикой (профили платформ, события, **группы**, тренерская и админская зоны, RBAC на уровне middleware и lib). Ветка **`group`** синхронизирована с **`origin/group`**; последние изменения по сообщению коммитов касаются тренерской зоны, групп и приватных событий в разрезе групп.

Документация сосредоточена в **`docs/`**; для операционной работы полезны `ENV_SETUP.md`, `INTERNAL_JOBS.md` и RBAC-файлы. CI нацелен на **`master`**, а не на текущую ветку — при слиянии в `master` проверки сработают автоматически.

| Статус | Пункт |

### 9.6. Как читать чеклист

**Ядро RBAC** (роли, матрица прав, middleware по ключевым зонам, guards на админ API и части тренерских API, клиентские guards, шапка, ИИ) — **реализовано**. **Пробелы:** смешанный стиль защиты API (часть эндпоинтов всё ещё на ручных `getServerSession` проверках вместо `withRoleGuard`), отсутствие тестов и непроверяемая из репозитория стадия миграций SurrealDB.
